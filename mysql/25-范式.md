
## 范式
```
可能的问题
* 数据冗余
* 插入, 删除, 更新异常
* 性能低


    超键: 唯一标识一行的属性集
  候选键: 超键中去掉多余的属性
    主键: 从候选键中选一个作为主键
    外键: 该键的值是另一表中
  主属性: 候选键中的属性
非主属性: 除主属性外的其他属性

# 0. 测试表
*   表: 学号 姓名 系名 系主任 课名 成绩
* 主键: (学号 课名)

# 1. 第一范式
每一个属性不可拆分

问题:
* 同一个学号, 不同的课名, 姓名重复 -------------- 冗余
* 同一个学号, 不同的课名, 系主任重复 ------------ 冗余
* 同一个学号, 不同的课名, 系名重复 -------------- 冗余
* 同一个系名, 系主任重复 ------------------------ 冗余
* 无学生数据时, 系信息也不存在 ------ 插入异常
* 删除所有学生后, 系信息也不存在 ---- 删除异常
* 系主任变化后, 相关列都得更新 ------ 更新多
* 学生姓名变化后, 相关列都得更新 ---- 更新多
* 查询 ------------------------------ 效率高

# 2. 第二范式: 消除了非主属性对于码的部分函数依赖
* 主键: (学号 课名)
*   系名由学号唯一确定, 和课名无关, 可以拆出来
* 系主任由学号唯一确定, 和课名无关, 可以拆出来
*   姓名由学号唯一确定, 和课名无关, 可以拆出来

修改:
拆表 (学号 课名 成绩)  (学号 姓名 系名 系主任)

问题:
* 同一个学号, 不同的课名, 姓名重复 -------------- 冗余 (解决)
* 同一个学号, 不同的课名, 系主任重复 ------------ 冗余 (解决)
* 同一个学号, 不同的课名, 系名重复 -------------- 冗余 (解决)
* 同一个系名, 系主任重复 ------------------------ 冗余 (未解决)
* 无学生数据时, 系信息也不存在 ------ 插入异常 (未解决)
* 删除所有学生后, 系信息也不存在 ---- 删除异常 (未解决)
* 系主任变化后, 相关列都得更新 ------ 更新多 (部分解决)
* 学生姓名变化后, 相关列都得更新 ---- 更新多 (部分解决)
* 查询 ------------------------------ 效率高 (可能降低)

# 3. 第三范式: 消除了非主属性对于码的传递函数依赖
*   表: (学号 姓名 系名 系主任)
* 主键: 学号
* 系主任和系名直接确定, 和学号是间接依赖, 可以拆出来

修改:
拆表 (学号 课名 成绩)  (学号 姓名 系名) (系名 系主任)

问题:
* 同一个学号, 不同的课名, 姓名重复 -------------- 冗余 (解决)
* 同一个学号, 不同的课名, 系主任重复 ------------ 冗余 (解决)
* 同一个学号, 不同的课名, 系名重复 -------------- 冗余 (解决)
* 同一个系名, 系主任重复 ------------------------ 冗余 (解决)
* 无学生数据时, 系信息也不存在 ------ 插入异常 (解决)
* 删除所有学生后, 系信息也不存在 ---- 删除异常 (解决)
* 系主任变化后, 相关列都得更新 ------ 更新多 (解决)
* 学生姓名变化后, 相关列都得更新 ---- 更新多 (部分解决)
* 查询 ------------------------------ 效率高 (可能降低)
```
