
## 介绍
IPv6 零压缩

## 目的
1. 提高可读性
2. 不容易写错
3. 同一个 IPv6 只有一种表示, 方便文本处理

## 算法
### 1. 删除每一段前置的0
#### 1.1 如果该段全部为0时, 该段删除后的结果是单个0
```
压缩前: 1234:1234:1234:1234:1234:1234:1234:0000
压缩后: 1234:1234:1234:1234:1234:1234:1234:0
```

#### 1.2 其他情况正常删除即可
```
压缩前: 1234:1234:1234:1234:1234:1234:1234:000a
压缩后: 1234:1234:1234:1234:1234:1234:1234:a
```

### 2. 压缩最长的连续字段为零的段
#### 2.1 只有一段为 0 时, 不压缩
```
压缩前: 1234:0:1234:1234:1234:1234:1234:1234
压缩后: 1234:0:1234:1234:1234:1234:1234:1234
```

#### 2.2 多段一样长时, 压缩第一段
```
压缩前: 1234:0:0:1234:0:0:1234:1234
压缩后: 1234::1234:0:0:1234:1234
```

#### 2.3 全部为0时, 结果是 ::
```
压缩前: 0000:0000:0000:0000:0000:0000:0000:0000
压缩后: ::
```

#### 2.4 其他情况压缩最长的
```
压缩前: 1234:0:0:1234:0:0:0:1234
压缩后: 1234:0:0:1234::1234
```
